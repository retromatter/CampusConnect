CampusConnect

Implementation Plan — Step-by-Step Build Guide

React Native + Expo React Native Paper Firebase 8 Phases School Project

Start here once all prerequisites are confirmed done.

How to use this plan: Follow phases in order. Each phase ends with a git commit. Do not start a new phase until the current one works and is committed. Phases 1–4 are basic features (no internet needed). Phases 5–7 are advanced features added one at a time.

ST Tech Stack

  • Framework React Native + Expo JavaScript — no Android Studio, no Java. QR scan to run on phone. Code is readable for class presentations.
  • UI React Native Paper Material Design for mobile. MUI equivalent — same Google-style components (Card, Button, FAB, TextInput). Works on React Native, MUI does not.
  • Navigation React Navigation Industry standard. Tab navigation for main screens, stack navigation for login flow.
  • Local Storage AsyncStorage Phases 1–4 only. Saves data on-device. No internet required.
  • Auth Firebase Auth Email/password login. Free. 3 lines of code to sign in. Covers "Login System" grading criteria.
  • Database Firebase Firestore NoSQL cloud DB. Free Spark plan. One function call to save data. Covers "Online Database" grading criteria.
  • API OpenWeatherMap Free weather API. One fetch() call. Covers "Internet Data (API)" grading criteria. Easy to explain in a demo.
  • Demo Expo Go + EAS Update Publish free to Expo servers. Share a QR link — anyone scans and opens the app. No APK install needed.
  • APK Build EAS Build Builds .apk from GitHub in the cloud. Free (30 builds/month). No Android Studio needed.

5 Phase 5 — Login System

Advanced #1

First advanced feature. Users must log in to use the app. Powered by Firebase Auth.

Step 1 — Create src/services/firebase.js
import { initializeApp } from 'firebase/app'; import { getAuth } from 'firebase/auth'; import { getFirestore } from 'firebase/firestore'; const firebaseConfig = { // paste your config from Firebase Console here }; const app = initializeApp(firebaseConfig); export const auth = getAuth(app); export const db = getFirestore(app);
Step 2 — Build screens
  • Create src/screens/LoginScreen.js — email + password fields, Login button, link to Register
  • Create src/screens/RegisterScreen.js — same fields, Register button
  • Use createUserWithEmailAndPassword for register
  • Use signInWithEmailAndPassword for login
Step 3 — Lock navigation behind login
  • In App.js: use onAuthStateChanged to watch login state
  • If no user → show Login/Register stack
  • If user logged in → show main tab navigation
Firebase config values are not secret on their own — they are safe to commit to GitHub for this project. For production apps you would use environment variables.
git commit -m "Phase 5: login system (Firebase Auth)"

6 Phase 6 — Online Database

Advanced #2

Second advanced feature. Data moves from the device to Firestore in the cloud.

  • Announcements: replace hardcoded array with a Firestore announcements collection. Any announcement added in the Firebase Console appears live in the app.
  • Schedule: move from AsyncStorage → Firestore, stored under the logged-in user's UID (users/{uid}/schedule)
  • Notes: same — move to Firestore under users/{uid}/notes
  • Use onSnapshot() for live updates — data refreshes without pulling down to reload
How user data isolation works: When a user logs in, Firebase gives them a unique uid. Their schedule and notes are saved under that uid in Firestore. Other users cannot see each other's data.
git commit -m "Phase 6: online database (Firestore)"

7 Phase 7 — Internet Data / API

Advanced #3

Third advanced feature. Fetches live weather data from OpenWeatherMap and displays it on a home card.

Step 1 — Create src/services/weatherService.js
const API_KEY = 'your_openweathermap_key'; export async function getWeather(city) { const res = await fetch( `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${API_KEY}&units=metric` ); return res.json(); }
Step 2 — Add weather card to home/dashboard
  • Call getWeather('your city') on screen load with useEffect
  • Display: city name, temperature, weather condition
  • Show a loading state while fetching
  • Use a React Native Paper Card for the widget
All phases done: 4 basic features + 3 advanced features = 7 features total. Test the full app end-to-end before deploying.
git commit -m "Phase 7: weather API widget"

D Deployment

Option 1 — Expo Go Live (Recommended)
Publish to Expo servers. Share one QR link. Anyone with Expo Go scans and opens the app instantly. Your laptop does not need to be on. Free.
Option 2 — EAS Build (APK)
Builds a real .apk file from GitHub in the cloud. Classmates download and install it. Free tier: 30 builds/month. No Android Studio needed. Free.
Option 3 — GitHub Release + APK
Attach the .apk from EAS Build to a GitHub Release tag. Clean, permanent download link from your repo. Free.
Deploy commands
# Option 1 — Expo Go QR link (instant, free) npx expo login eas update --branch main --message "v1.0 release" # Option 2 — Build APK (free, builds from GitHub) npm install -g eas-cli eas build:configure eas build -p android --profile preview # After build: download .apk link from EAS dashboard # Option 3 — Attach .apk to GitHub Release # Repo → Releases → New Release → tag: v1.0 → attach .apk
Firebase (Auth + Firestore) is not "deployed" — it is already live in the cloud the moment you created the Firebase project. No extra steps needed.
