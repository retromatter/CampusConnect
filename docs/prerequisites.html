Note on Build Timeline

This document covers setup only. The build log and implementation timeline (phase-by-phase commits) will begin separately once all prerequisites below are confirmed done. No app code is written yet.

Why This Tech Stack — Reasoning

Context: This is a school project. The code will be presented to classmates and a teacher. Every decision below prioritizes readability, simplicity, and ease of demo over advanced architecture. You have more advanced setups available — this project is intentionally kept approachable.
Framework — React Native + Expo

React Native uses JavaScript — a language most students already know. Expo removes the need for Android Studio, Java, or any complex native setup. Running the app is one command: npx expo start. For the class demo, you scan a QR code and the app appears on a real phone instantly. The code stays short and readable — a classmate can look at a screen file and understand what it does.

UI — React Native Paper (Material Design)

MUI is the popular Material Design library for React web — it does not work on React Native. React Native Paper is its direct equivalent for mobile: same Material Design style, same component names (Button, Card, FAB, TextInput), same clean Google-like aesthetic you see on firebase.google.com. Pre-built components mean less custom CSS and more time on actual features.

Auth + Database — Firebase (Auth + Firestore)

Firebase requires no server to set up or maintain. Login is 3 lines of code. Saving data to the cloud is one function call: collection().add(data). The free Spark plan covers everything this project needs — no credit card, no billing. During the presentation you can show a user logging in and data appearing in the Firebase Console in real time. That is a clear, visual proof of both "Login System" and "Online Database" from the grading criteria.

API — OpenWeatherMap

A single free fetch() call returns live weather data for any city. No complex setup, no paid tier needed. On the home screen it shows temperature and conditions — a clear, visible demonstration of "Internet Data (API)" from the grading criteria. Easy to point to during a presentation and explain in one sentence.

Deployment — Expo Go + EAS Update

Publishing via eas update puts your app on Expo's servers for free. Anyone with the Expo Go app scans one QR code and the app opens — no APK install, no Android Studio. If a standalone APK is required, EAS Build generates one from your GitHub repo in the cloud. Everything is free, everything starts from GitHub.

Note (not planned yet): The PDF assignment also allows Android Studio (Java/Kotlin) and Flutter (Dart). Both are valid tools — skipped here because they require heavier setup and use languages less familiar to most classmates. See the "Alternative Tools" card below for details.

1 Install Node.js

nodejs.org

Node.js is required to run Expo and install packages. Download the LTS version.

  • Go to nodejs.org and download the LTS installer
  • Run the installer — next → next → finish
  • Open a terminal and verify it works
Verify install
node --version # Should print: v20.x.x or higher npm --version # Should print: 10.x.x or higher

2 Install Expo CLI

npm

Expo lets you build React Native apps without Android Studio. One command does everything.

  • Open terminal (Command Prompt or PowerShell)
  • Run the install command below
  • Verify it works with the check command
# Install Expo CLI globally npm install -g expo-cli # Verify expo --version
Why Expo? No Android Studio, no Java, no Gradle. Run one command, scan a QR code, and your app appears on your real phone. Perfect for class demos.

3 Install Expo Go on Your Phone

Play Store / App Store

Expo Go runs your code on a real phone — no APK install needed. This is how you demo to your class.

  • Android: search "Expo Go" on Google Play Store → Install
  • iPhone: search "Expo Go" on Apple App Store → Install
  • Open the app once to confirm it works
During development your phone and laptop must be on the same Wi-Fi network to scan the QR code.

4 GitHub — Account + Repository

github.com

All code lives here. Deployment (EAS Build) also reads from GitHub.

  • Create a free account at github.com if you don't have one
  • Create a new repository named campus-connect
  • Set visibility to Public — teacher can see the code
  • Do not initialize with a README yet — Expo creates the structure
  • Copy the repo URL — needed when pushing the first commit
Git setup (first time only)
git config --global user.name "Your Name" git config --global user.email "your@email.com"

5 Firebase — Create a Project

firebase.google.com

Firebase gives you a free login system and free cloud database. Only needs a Google account.

  • Go to firebase.google.com
  • Click "Get Started" → sign in with Google
  • Click "Create a project"
  • Name it: campus-connect
  • Google Analytics: disable it — not needed
  • Click "Create project" → wait → Continue
Spark Plan = Free Forever. No credit card, no billing. Supports 50,000 database reads/day and 10 simultaneous users — more than enough for a school demo.
Register your app with Firebase
  • Inside your project click the </> (Web) icon
  • App nickname: CampusConnect
  • Do not enable Firebase Hosting
  • Click Register app
  • Save the firebaseConfig object — paste it into the app in Phase 5
The config will look like this — save it to a text file for now:
// Save this — needed in Phase 5 const firebaseConfig = { apiKey: "your-api-key", authDomain: "campus-connect.firebaseapp.com", projectId: "campus-connect", storageBucket: "campus-connect.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" };

6 Firebase Auth — Enable Email/Password Login

Firebase Console

Powers the Login screen in Phase 5. Takes about 30 seconds to enable.

  • Firebase Console → left sidebar → Build → Authentication
  • Click "Get started"
  • Under Sign-in method tab → click Email/Password
  • Toggle the first switch to Enable
  • Click Save
Email/Password is enough — no need for Google Sign-In or phone auth. Simplest to explain in a presentation.

7 Firestore — Create the Database

Firebase Console

Firestore is the online database used in Phase 6 — stores announcements, schedules, and notes in the cloud.

  • Firebase Console → left sidebar → Build → Firestore Database
  • Click "Create database"
  • Choose Start in test mode — allows read/write during development
  • Location: pick the closest region — e.g. asia-southeast1 for Philippines
  • Click Enable
Test mode allows anyone to read/write for 30 days. That is fine for a school demo — security rules can be tightened later.

8 OpenWeatherMap — Get a Free API Key

openweathermap.org

Used in Phase 7 to show live weather on the home screen. Covers the "Internet Data (API)" grading criteria.

  • Go to openweathermap.org → Sign In → Create an Account
  • Verify your email
  • Go to your profile → My API keys
  • Copy the default key — looks like: a1b2c3d4e5f6...
  • Save the key in a text file for later
Wait up to 2 hours after creating an account before the key becomes active. Do this step early.
Test your key — paste this in a browser (replace YOUR_KEY)
https://api.openweathermap.org/data/2.5/weather?q=Manila&appid=YOUR_KEY&units=metric

If you see JSON with temperature info → key is working.

9 VS Code — Code Editor

code.visualstudio.com

The editor you'll write code in. Free from Microsoft.

  • Download from code.visualstudio.com → install
  • Install extension: ES7+ React/Redux/React-Native snippets
  • Install extension: Prettier - Code formatter
  • Optional: React Native Tools by Microsoft

Noted: Alternative Tools (Not Planned Yet)

The PDF assignment allows these tools. Noted here for reference — no plans to switch unless decided later.

Android Studio

Uses Java or Kotlin. Heavy setup — JDK, SDK manager, emulator config. Harder to present to classmates. Valid for a fully native APK.

Flutter

Uses Dart language. Great performance and UI. Also requires Android Studio setup. Dart is less familiar to most students than JavaScript.

Why React Native + Expo instead: JavaScript-based (most students know it), no Android Studio needed, demo by scanning a QR code on any phone, and the code is short and readable enough to walk classmates through.
